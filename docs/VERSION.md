## 1. 현재 버전

- 현재 버전: `v0.10.0`
- 상태: Public MVP / Beta
- 기준일: 2026-05-14
- 단계: 공개 MVP 이후 첫 주요 기능 추가 릴리즈

`v0.10.0`은 경기별 커뮤니티 기능인 경기톡을 추가한 베타 기능 확장 버전입니다. 기존 직관 후기 중심의 커뮤니티에, 사진이나 직관 인증 없이도 특정 KBO 경기에 가볍게 반응할 수 있는 게시판 흐름을 더했습니다.

아직 정식 `v1.0.0`으로 보지 않는 이유는 운영 관측성, 법무/동의 보강, 운영 문서화, 첫 사용자 피드백 반영이 남아 있기 때문입니다.

## 2. 버전 정책

프로젝트는 SemVer를 가볍게 따르되, 제품 단계에 맞춰 아래 기준으로 운영합니다.

- `v0.x`: MVP, 베타, 사용자 피드백 반영 단계
- `v0.9.x`: 공개 MVP 이후 버그 수정, 운영 안정화, 작은 UX 개선
- `v0.10.x`: 공개 MVP 이후 의미 있는 기능 추가 또는 주요 UX 개선
- `v1.0.0`: 정식 출시 기준 충족

변경 단위:
- patch: 버그 수정, 문구 수정, 작은 UI/UX 개선, 운영 설정 보강
- minor: 새 기능 추가, 주요 사용자 흐름 개선, 운영 기능 추가
- major: 큰 제품 방향 변경, 데이터 구조나 호환성에 영향을 주는 변경

## 3. v1.0.0 기준

아래 항목이 충분히 정리되면 `v1.0.0` 정식 출시로 올릴 수 있습니다.

- 실제 사용자 피드백 1차 반영
- Vercel/Supabase/KBO 동기화 운영 로그 안정 확인
- 이용약관/개인정보처리방침 법무 검토
- 회원가입 또는 계정 전환 시 동의 체크박스 도입
- 환경변수와 seed/SQL 적용 순서 문서화
- README 실행/배포/운영 기준 보강
- 치명적인 핵심 흐름 이슈 없음

## 4. 릴리즈 기록

### v0.10.0 - 2026-05-14

경기톡 기능 추가 릴리즈.

추가 기능:
- 커뮤니티에 [후기] / [경기톡] 서브탭 추가
- 특정 KBO 경기에 묶이는 경기톡 글 작성, 조회, 삭제
- 로그인 사용자의 경기톡 작성 지원
- 비로그인 사용자의 경기톡 목록 조회 허용
- 본문 300자 제한, 감정 태그 1개 선택
- 사진 0~1장 첨부 지원
- 작성 시점의 경기 점수와 상태를 글에 저장
- 경기톡 좋아요와 댓글 지원
- 후기 댓글 UI를 공용 `CommentThread` 흐름으로 정리해 경기톡에서도 재사용
- 경기톡 카드에 응원팀, 직관 인증, 점수, 경기 상태, 좋아요 수, 댓글 수 표시
- 홈의 이번 주 경기 카드에서 경기톡 글 수 배지 표시
- 일정 화면에서 해당 경기의 경기톡으로 이동하는 링크 추가
- 경기톡 전용 DB 테이블과 RLS 정책 추가
- 라이브 스코어 lazy refresh 기준 추가

공지사항 요약:
- "경기별로 모여 이야기하는 경기톡이 추가되었습니다."
- "경기 중 응원, 분노, 불안, 환호 같은 감정을 짧게 남길 수 있습니다."
- "홈과 일정에서 바로 해당 경기의 경기톡으로 이동할 수 있습니다."

운영 확인:
- 경기톡 기획 문서 검토 완료
- 경기톡 개발 계획 기준 Step 0~9 완료 기록 확인
- `feature/match-talk` master 병합 및 운영 배포 완료 기록 확인

### v0.9.0 - 2026-05-13

Public MVP / Beta.

포함 기능:
- 랜딩, 로그인, 온보딩
- Google, Kakao, 이메일, 익명 로그인
- 익명 사용자의 정식 계정 전환 흐름
- 팀 기반 홈 화면과 직관 승률
- KBO 일정/순위 연동
- 직관 등록과 예매권 인증
- 미인증 직관 사후 인증
- 경기 종료 결과 확인 모달
- 후기 작성, 후기 상세, 사진 캐러셀, 라이트박스
- 좋아요, 저장, 댓글
- 커뮤니티 피드와 친구 필터
- 친구 검색, 요청, 수락, 거절, 삭제
- 공유 카드 이미지 생성과 Web Share/download fallback
- 마이 페이지, 내 직관, 내 후기, 저장한 후기
- 공지, 이용안내, 문의, 이용약관, 개인정보처리방침
- PWA 설치 안내, Pull-to-Refresh, visibility refresh

운영 백로그:
- 운영 관측성 강화
- 법무/동의 보강
- 환경변수와 seed 문서화
- README 보강
- Supabase Realtime 검토
- 직관 경험치/레벨 시스템 검토
- 푸시 알림 검토
